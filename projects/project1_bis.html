French Book Data Collection Process

This project involves scraping book details from a specified online source. The goal is to collect comprehensive information about each book in french, including title, author, publication date, editor, rating, summary, and genre, as well as images for visual representation. The steps below detail the data collection process used :

Project Goals

  • Building a dataset with french book information

Data Collection Methodology

  • Page Navigation: The script first retrieves all book pages by accessing paginated URLs, which contain multiple books per page. Each page's HTML is parsed to gather individual book URLs.
  • Data Extraction: For each book URL, the code extracts:
    • Title and Author: The title and author are located within specific HTML tags. The script handles variations in formatting by using multiple strategies to locate the data, improving data reliability.
    • Release Date, Editor, and EAN: These details are collected from metadata sections on each book's page, and only the release year is extracted from the full date format.
    • Rating: The script calculates an approximate rating based on filled star elements on the page.
    • Summary: The book summary is retrieved from a structured content block, and special characters are removed to create a cleaner format.
    • Genre: The genre is retrieved once per main URL page, representing the category of all books on that page.
  • Image Processing: Each book's cover image is retrieved and resized for optimized display on different parts of the application. Images are encoded in base64 format to be used directly in the application.
  • Error Handling: The script includes robust error-handling mechanisms to manage missing data or connection issues. Each extraction step is wrapped in a try-except block to log any errors encountered and avoid interruption of the scraping process.
  • Data Caching and Restart: The program caches the URLs already processed to avoid re-scraping. It also supports resuming from where it last left off, which is useful in case of interruptions.

Data Post-Processing

After the initial data collection, post-processing steps were implemented to ensure data quality and usability:

  • Data Cleaning: Removed any duplicates and entries with missing critical information such as titles or authors. This step helps maintain the integrity of the dataset.
  • Final Dataset Creation: Compiled all the processed data into a final CSV file, ready for analysis or presentation.

The data collected is then saved in a CSV format, allowing for easy analysis and visualization in subsequent parts of the project.
